## **PROMPT FOR AGENT**

Build a complete Android application in **Java (not Kotlin)** that records video in a **privacy-compliant and policy-compliant manner**.

---

## **App Overview**

* App name: **"Video Recorder"**
* The app allows the user to record video using the device camera
* Recording must always be **transparent and visible to the user**
* Must comply with Android 8+ (API 26 and above) background execution limits

---

## **Core Features**

### 1. Video Recording

* Use **Camera2 API** (preferred) or CameraX (Java-compatible usage only)
* Record video using device camera
* Save video files to:

  ```
  /sdcard/Downloads/vdo/
  ```
* Ensure directory is created if it does not exist
* Use proper file naming (e.g., `VID_yyyyMMdd_HHmmss.mp4`)

---

### 2. Start / Stop Controls

* Main screen must include:

  * **Start button**
  * **End button**
* Behavior:

  * Start:

    * Starts video recording
    * Starts a **Foreground Service**
  * End:

    * Stops recording
    * Stops the service

---

### 3. Foreground Service (MANDATORY)

* Recording must run inside a **Foreground Service**
* Show a persistent notification:

  * Example text: "Recording in progress"
* Notification must:

  * Be visible at all times during recording
  * Include an optional action to stop recording

---

### 4. User Transparency (REQUIRED)

* Show clear indication in UI:

  * Recording status (Recording / Stopped)
* Optionally show:

  * Camera preview on screen (recommended)
* Do NOT hide recording behavior

---

### 5. Permissions

Handle runtime permissions:

* `CAMERA`
* `RECORD_AUDIO`
* `WRITE_EXTERNAL_STORAGE` (for API < 29)
* For API 29+ use **MediaStore or scoped storage**

Gracefully handle:

* Permission denial
* Camera unavailable

---

### 6. File Storage

* Save videos in public user-accessible directory:

  * `/Downloads/vdo/`
* For Android 10+:

  * Use **MediaStore API**
* Ensure files are visible in gallery/file manager
